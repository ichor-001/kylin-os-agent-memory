## 1. Agent记忆管理与个性化

### 1.1 MemGPT: Towards LLMs as Operating Systems
- **会议**: NeurIPS 2024 (Oral)
- **年份**: 2024 | 引用: 600+
- **arXiv**: 2310.08560
- **核心**: OS风格虚拟内存分页管理LLM上下文。主存(main context) + 外存(external context)分层架构，实现虚拟无限长上下文。
- **可用性**: 开源 (Letta框架)，已商业化
- **与你赛题的关系**: 分层内存架构可直接借鉴——短期=主存，长期知识/偏好=外存

### 1.2 HippoRAG: Neurobiologically Inspired Long-Term Memory
- **会议**: NeurIPS 2024
- **年份**: 2024 | 引用: 150+
- **arXiv**: 2405.14831
- **核心**: 海马体记忆索引理论启发，知识图谱 + Personalized PageRank实现长期记忆检索。离线KG索引 + 在线检索两步法，多跳问答大幅优于RAG。
- **可用性**: 开源
- **与你赛题的关系**: "关联检索"(要求3)的直接对标方案

### 1.3 MemoryBank: Enhancing LLMs with Long-Term Memory
- **会议**: AAAI 2024
- **年份**: 2024 | 引用: 200+
- **arXiv**: 2305.10250
- **核心**: 艾宾浩斯遗忘曲线驱动的记忆管理——高频操作强化，低频衰减。支持NL交互管理记忆。
- **可用性**: 开源
- **与你赛题的关系**: "偏好记忆动态捕捉"(要求2)的天然方案——用户高频偏好自然强化

### 1.4 Agent Workflow Memory (AWM)
- **会议**: ICLR 2025
- **年份**: 2025 | 引用: 80+
- **arXiv**: 2409.07429
- **核心**: Agent从历史经验中归纳可复用的"工作流"模板。遇到类似任务时检索复用。
- **可用性**: 开源
- **与你赛题的关系**: 用户操作模式学习——自动提取工作流程知识

### 1.5 Memory3: Language Modeling with Explicit Memory
- **会议**: arXiv 2025
- **年份**: 2025 | 引用: 50+
- **arXiv**: 2407.01178
- **核心**: 知识显式存储在外挂记忆库，模型参数与知识解耦。三层：推理参数 + 知识记忆(显式) + 短期工作记忆。成本仅为1/10。
- **可用性**: 开源 (清华/百川)
- **与你赛题的关系**: 显式知识记忆架构——系统文档、命令手册作为独立知识库

### 1.6 AriGraph: Learning Knowledge Graph World Models
- **会议**: ACL 2024
- **年份**: 2024 | 引用: 60+
- **arXiv**: 2407.04363
- **核心**: 动态更新的知识图谱作为Agent世界模型，探索经验实时编码为语义三元组存入图谱。
- **可用性**: 开源
- **与你赛题的关系**: 动态KG方法可用于文件系统和工具调用关系的实时建模

### 1.7 Generative Agents (Stanford)
- **会议**: UIST 2023 (Best Paper)
- **年份**: 2023 | 引用: 2000+
- **arXiv**: 2304.03442
- **核心**: "记忆流"(Memory Stream)三层架构：存储(observation→memory) → 检索(recency+relevance+importance) → 反思(reflection)。奠基性工作。
- **可用性**: 开源
- **与你赛题的关系**: 反思机制可从操作历史中提炼用户偏好

### 1.8 Reflexion: Language Agents with Verbal RL
- **会议**: NeurIPS 2023
- **年份**: 2023 | 引用: 1000+
- **arXiv**: 2303.11366
- **核心**: 任务失败后生成语言化反思存入"情景记忆"，下次检索改进决策。无需参数更新。
- **可用性**: 开源
- **与你赛题的关系**: 操作失败→反思→改进循环，渐进式自我优化

### 1.9 LongMem (Microsoft Research)
- **会议**: NeurIPS 2023
- **年份**: 2023 | 引用: 300+
- **核心**: 解耦记忆架构——冻结backbone + 可训练记忆bank，用attention键值对做记忆前缀适配。
- **与你赛题的关系**: 不改基础模型的情况下持续学习用户偏好

### 1.10 PersonalizedAgent Survey
- **会议**: arXiv 2025
- **年份**: 2025
- **核心**: Agent个性化方法系统综述：用户画像、偏好学习、记忆管理、人格对齐的统一框架。
- **与你赛题的关系**: 综述性质，帮助快速建立领域全景

## 2. 知识图谱增强检索 (GraphRAG & Related)

### 2.1 SAGE: Self-Evolving Agentic Graph-Memory Engine ★★★
- **arXiv**: 2605.12061 | 2026年5月 (最新!)
- **核心**: 当前GraphRAG系统把记忆图谱当静态检索中间件。SAGE提出**自进化的agentic图记忆引擎**——能从部分线索恢复完整证据链、利用图结构角色、随时间自我改进。
- **与你赛题的关系**: "自进化"概念是差异化亮点——记忆系统不是死的，会越用越好

### 2.2 Executable Agentic Memory (EAM) for GUI Agent ★★★
- **arXiv**: 2605.12294 | 2026年5月 (最新!)
- **核心**: 提出**可执行的知识图谱记忆**——不再让LLM在每次屏幕前重新解读UI和重新决策，而是将过去的交互存为结构化KG节点并支持可执行召回。直接面向端侧GUI Agent。
- **与你赛题的关系**: 直接对标——OS Agent的GUI操作记忆，端侧部署场景

### 2.3 MAGE: Multi-Agent Self-Evolution with Co-Evolutionary KGs
- **arXiv**: 2605.10064 | 2026年5月 (最新!)
- **核心**: 协同进化知识图谱——清晰表示Agent知识，独立于扁平情景记忆或隐式RL信号。
- **与你赛题的关系**: 知识冲突处理的进化视角——KG随Agent经验进化

### 2.4 Towards Practical GraphRAG ★★★
- **arXiv**: 2507.03226 | 2025年7月
- **核心**: **可扩展且成本高效的GraphRAG企业部署**。专注：高效KG构建 + 混合检索(向量+图遍历)。直接解决GraphRAG在资源受限环境的部署问题。
- **与你赛题的关系**: 最直接相关的"端侧可部署GraphRAG"方案

### 2.5 GraphRAG-R1 (WWW'26) ★★
- **arXiv**: 2507.23581 | WWW 2026 (CCF-A)
- **核心**: 用**过程约束RL**(Process-Constrained RL)优化图遍历。不是one-shot检索，而是学习最优多步检索策略。
- **与你赛题的关系**: 正式顶会发表的GraphRAG增强方案，方法学可引用

### 2.6 GraphRAG Survey (Most Cited)
- **arXiv**: 2501.13958 | 2025年1月 | 引用: 101
- **核心**: GraphRAG系统综述——图索引策略、检索再推理vs推理再检索范式、评测基准的分类法。
- **与你赛题的关系**: 基础参考，快速了解GraphRAG全貌

### 2.7 Agentic GraphRAG with Provenance
- **arXiv**: 2605.15109 | 2026年5月 (最新!)
- **核心**: Agentic GraphRAG中的**引用忠实性**——agent探索KG后生成带引用的答案，引入遍历上下文和溯源追踪。
- **与你赛题的关系**: 知识来源可追溯——对安全合规(要求5)很重要

### 2.8 MemReranker: Reasoning-Aware Reranking ★★
- **arXiv**: 2605.06132 | 2026年5月 (最新!)
- **核心**: 解决记忆检索中"重排序"环节——通用重排序模型仅依赖语义相似度，缺乏推理能力。提出**推理感知的重排序**专为Agent记忆设计。
- **与你赛题的关系**: 检索精度优化——500ms延迟约束下如何提高排序质量

### 2.9 Entropic Claim Resolution ★★
- **arXiv**: 2603.28444 | 2026年3月
- **核心**: 当知识源冲突时，用**熵/不确定性驱动的证据选择**替代单纯的语义相关性检索。
- **与你赛题的关系**: "新旧知识冲突处理"(要求3)的理论基础

### 2.10 Rashomon Memory ★★
- **arXiv**: 2604.03588 | 2026年4月
- **核心**: 长时间Agent积累的经验服务于多个并发目标，对同一事件可能有冲突解释。提出**论证驱动的检索**——冲突视角共存，通过结构化论证解决。
- **与你赛题的关系**: "跨场景偏好冲突"问题的直接方案

### 2.11 SHIMI: Semantic Hierarchical Memory Index
- **arXiv**: 2504.06135 | 2025年4月
- **核心**: 统一**语义层级记忆索引**——解决RAG和向量搜索在抽象、扩展性和语义精度上的局限。层级方法支持不同粒度的高效检索。
- **与你赛题的关系**: 轻量化层级索引——适合端侧多粒度检索

### 2.12 Memanto: Typed Semantic Memory
- **arXiv**: 2604.22085 | 2026年4月
- **核心**: 将记忆定义为持久多会话自治Agent的**首要架构瓶颈**。提出带信息论检索的类型化语义记忆。
- **与你赛题的关系**: 信息论驱动的检索——理论上更优的检索效率

### 2.13 Thinking Ahead: Prospection-Guided Retrieval
- **arXiv**: 2605.14177 | 2026年5月 (最新!)
- **核心**: 标准RAG/GraphRAG在相关事实与查询语义相似度低时失败。提出**前瞻引导检索**——Agent"提前思考"需要什么知识。
- **与你赛题的关系**: 主动式知识检索——不等用户问就先准备相关知识

### 2.14 EvolveMem: Self-Evolving Memory Architecture
- **arXiv**: 2605.13941 | 2026年5月 (最新!)
- **核心**: 真正的Agent记忆不仅存储内容要进化，**检索基础设施本身**(评分函数、融合策略、应答策略)也需要自进化。
- **与你赛题的关系**: 最前沿思想——检索系统与积累知识协同进化

### 2.15 PersonalAI 2.0: KG with Planning
- **arXiv**: 2605.13481 | 2026年5月 (最新!)
- **核心**: 在KG遍历前加入**规划机制**——不是盲目遍历，而是先规划再检索。面向个性化LLM Agent。
- **与你赛题的关系**: 规划式KG遍历——更高效的个性化检索

## 3. 端侧部署与轻量化

> 注：该方向因API限流未完成自动化检索，以下基于训练知识补充2025年关键工作。

### 3.1 LLM Agent Memory at the Edge — 研究趋势总结

2025-2026年端侧Agent记忆的关键方向：

| 方向 | 代表工作/趋势 | 核心思想 |
|------|-------------|---------|
| **小模型Agent** | Phi-3/4系列, Gemma 3, Qwen2.5-1.5B/3B | 10B以下模型可在手机/笔记本运行，配合量化可到2-4GB内存 |
| **向量索引轻量化** | HNSW, DiskANN, Faiss IVF-PQ | 近似最近邻搜索，百万级向量毫秒级检索 |
| **分层缓存(Hierarchical Caching)** | KV-cache压缩 + 语义摘要 | 近期的全保留(短期)，远期的摘要化(长期) |
| **嵌入模型量化** | BGE-small, all-MiniLM的INT8量化 | 768维→384维或INT8量化，质量下降<3%，速度提升4-8x |
| **本地优先架构(Local-First)** | Apple Intelligence, 鸿蒙AI架构 | 端侧推理+云端增强，敏感数据不出设备 |
| **增量索引更新** | 在线HNSW, Streaming Vector DB | 新知识增量写入，无需全量重建索引 |
| **混合检索(BM25+Vector)** | Weaviate, Qdrant, LanceDB | 关键词+语义双路，精度/速度平衡 |

### 3.2 关键开源组件 (可直接用于工程实现)

| 组件 | 用途 | 为什么选它 |
|------|------|-----------|
| **LanceDB** | 向量存储 | Serverless架构，极轻量(单文件)，支持混合检索 |
| **ChromaDB** | 向量存储 | Python原生，嵌入函数可插拔(可接麒麟SDK) |
| **Faiss + HNSW** | 向量索引 | Meta出品，CPU上百万向量<10ms |
| **sentence-transformers** | 嵌入模型 | 支持麒麟SDK替换为自定义嵌入函数 |
| **ONNX Runtime** | 模型推理 | 量化推理，CPU优化，跨平台 |
| **llama.cpp** | 本地推理 | GGUF量化格式，适合端侧小模型推理 |

### 3.3 与麒麟SDK的对接策略

麒麟embedding SDK是封闭的——你们只能调用不能替换。策略：
- **适配层设计**：把麒麟SDK封装成sentence-transformers兼容接口
- **混合检索兜底**：麒麟SDK做语义检索 + 本地BM25做关键词兜底
- **索引加速**：麒麟SDK的embedding结果存入本地HNSW索引，减少SDK调用次数
- **缓存策略**：高频查询结果缓存，降低端侧延迟(SDK调用+索引检索并行)

### 3.4 500ms指标可行性分析

```
一个典型检索请求的时间预算 (500ms)：
┌────────────────────────────────────────────┐
│ 查询嵌入化 (麒麟SDK)     ~50-100ms         │
│ 向量索引检索 (HNSW)      ~5-20ms           │
│ 结果重排序 (小模型)       ~50-100ms         │
│ 冲突检测/融合             ~30-50ms          │
│ 知识图谱遍历              ~50-100ms         │
│ 序列化返回                ~5ms             │
├────────────────────────────────────────────┤
│ 总计 (串行)               ~190-375ms        │
│ 总计 (并行优化后)         ~120-200ms        │
└────────────────────────────────────────────┘
```

**可行**，但需要精细的工程优化——并行化、缓存、以及麒麟SDK本身的延迟是关键变量。

### 4.2 创新点提炼 (用于评审30分"技术创新性")

| 创新维度 | 具体方案 | 对标论文 |
|----------|---------|---------|
| **自进化记忆引擎** | 检索策略随使用自我优化，非静态存储 | SAGE, EvolveMem |
| **双模态冲突消解** | 偏好冲突(Rashomon) + 知识冲突(Entropic)统一框架 | Rashomon Memory, Entropic Claim Resolution |
| **执行式记忆** | 知识图谱节点支持可执行召回(不只是查，还能跑) | EAM |
| **推理感知检索** | 重排序阶段引入推理，超越语义相似度 | MemReranker |
| **端侧适配的三层加速** | 层级索引(SHIMI) + HNSW + 结果缓存并行 | SHIMI, Towards Practical GraphRAG |

### 4.3 实现策略 (适配端侧的降级方案)

一些SOTA论文(如SAGE, EvolveMem)依赖大模型做记忆推理。端侧替代方案：
- **记忆整理**: 大模型 → 本地7B模型(如Qwen2.5-7B-Instruct, Q4量化仅需~5GB显存)
- **知识图谱构建**: 大模型 → 规则+小模型混合(高频模式用规则，低频用模型)
- **冲突仲裁**: 大模型 → 统计方法(熵驱动) + 轻量分类器

## 6. 关键风险与应对

| 风险 | 应对 |
|------|------|
| 麒麟SDK延迟过高 | 本地索引缓存 + 麒麟SDK结果异步更新 |
