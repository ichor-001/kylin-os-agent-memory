## 一、最炸的5篇——可以直接作为创新出发点的论文

### ⭐ 1. Agentic Memory (AgeMem) —— 2026

| 信息 | 内容 |
|------|------|
| **arXiv** | 2601.01885 |
| **引用** | 16（刚出不久，热度很高） |
| **核心贡献** | 统一LTM+STM管理框架，**LLM Agent自主决定**存什么、何时存、何时检索——不依赖外部规则 |

**为什么炸：**
传统记忆系统靠人工规则（"超过5轮对话就摘要"、"相似度>0.8就合并"）。AgeMem把记忆管理本身变成一个**学习任务**，Agent通过强化学习自己学会记忆策略。

**对赛题的创新启发：**
```
赛题要求(2)「偏好动态捕捉」→ 让Agent自主判断「这个偏好值得记吗」
赛题要求(3)「知识冲突处理」→ 让Agent自己决定「新知识覆盖旧知识，还是并存」
→ 30分技术创新性的突破口
```

### ⭐ 2. Zep: Temporal Knowledge Graph Architecture for Agent Memory —— 2025

| 信息 | 内容 |
|------|------|
| **arXiv** | 2501.13956 |
| **引用** | 143（很高） |
| **核心贡献** | 时序知识图谱作为Agent记忆层，在Deep Memory Retrieval基准上**超越MemGPT** |

**为什么炸：**
传统记忆系统用向量数据库存"记忆碎片"，Zep用**时序知识图谱**——每个记忆不仅是向量，还有时间戳和关系链接。"上周用户的设置习惯"和"今天的设置习惯"在图上是有路径可查的。

**对赛题的创新启发：**
```
赛题要求(3)「关联检索」→ 不是找"相似的"，而是找"有关联的"
赛题要求(2)「版本化管理」→ 时序KG天然支持版本回溯
→ 可替代传统的向量数据库方案，作为技术架构创新
```

### ⭐ 3. ARTEM: Spatial-Temporal Episodic Memory —— AAAI 2026

| 信息 | 内容 |
|------|------|
| **会议** | **AAAI 2026**（CCF-A类顶级会议） |
| **核心贡献** | 混合LLM Agent架构，集成空间-时间情景记忆的自我组织神经网络 |

**为什么炸：**
AAAI 2026的正式论文。ARTEM不只用向量存记忆，而是用**自组织神经网络**来聚类和组织记忆——类似人脑的海马体如何组织情景记忆。这比传统的"对话历史→向量→检索"三层流水线更接近生物智能。

**对赛题的创新启发：**
```
可以把ARTEM的情景记忆思想移植到OS Agent场景：
用户的操作序列 → 自组织聚类 → 形成"场景记忆块"
跨场景切换时，自动激活对应记忆块
→ 要求(2)跨场景偏好适配的创新方案
```

### ⭐ 4. CatRAG (Context-Aware Traversal for RAG) —— 2026
#### 是 HippoRAG 的后续工作

| 信息 | 内容 |
|------|------|
| **arXiv** | 2602.01965 |
| **引用** | 2（刚出） |
| **核心贡献** | 静态KG→**查询自适应动态导航**，在HippoRAG 2架构上改进 |

**为什么重要：**
HippoRAG（NeurIPS 2024）本身就是知识图谱增强检索的标杆。CatRAG把它升级了——不再是建好KG就不动了，而是**每次查询时动态调整KG的导航结构**。

**对赛题的创新启发：**
```
赛题要求(3)「关联检索优化」→ 动态KG vs 静态KG
端侧的KG不可能太大，动态导航可以在小KG上实现高召回
→ 契合要求(4)端侧500ms约束
```

### ⭐ 5. SRAS: Lightweight RL-based Document Selector for Edge-Native RAG —— 2025

| 信息 | 内容 |
|------|------|
| **会议** | ICEdge 2025 |
| **arXiv** | 2601.01785 |
| **核心贡献** | **首个**用RL做轻量文档选择的端侧RAG方案，延迟感知 |

**为什么重要：**
赛题的500ms硬约束是最大技术挑战之一。SRAS给出了一条明确的路线——用**强化学习**训练一个轻量选择器，在保证召回率的前提下大幅降低搜索延迟。

**对赛题的创新启发：**
```
赛题要求(4)「检索响应≤500ms」→ 两级索引：
  第一级：RL训练的轻量选择器（毫秒级）
  第二级：麒麟SDK的embedding精排
→ 可独立作为一个技术创新点发表
```

## 三、你的创新武器库：7个可选的创新方向

综合以上论文，以下是可以打"技术创新性30分"的7个方向，从易到难：

### 方向A：Agent自主记忆管理（基于AgeMem的思想）
**难度**：高 | **创新性**：⭐⭐⭐⭐⭐ | **实现可行性**：中

把记忆管理变成LLM Agent的一个内在能力——Agent自主判断什么要记住、什么该遗忘、新旧知识冲突时怎么处理。你可以用LLM（本地小模型）+ 轻量规则实现一个简化版。

### 方向B：时序知识图谱记忆层（基于Zep的思想）
**难度**：中高 | **创新性**：⭐⭐⭐⭐ | **实现可行性**：高

你的TencentDB已经有四层记忆pipeline，可以在L1（事实提取）之上加一层时序KG——每条记忆带时间戳和关联边。用NetworkX + SQLite就能实现。

### 方向C：轻量RL检索选择器（基于SRAS的思想）
**难度**：中 | **创新性**：⭐⭐⭐⭐ | **实现可行性**：高

在麒麟embedding SDK搜索之前，加一个RL训练的轻量过滤器，先把候选集从万级缩小到百级，再走embedding精排。可以保证500ms约束。

### 方向D：动态构建的轻量知识图谱（基于Adaptive Graph RAG的思想）
**难度**：高 | **创新性**：⭐⭐⭐⭐⭐ | **实现可行性**：低

不预建完整的KG，而是每次查询时动态提取相关实体和关系，构建小KG用于检索。理论优雅但实现难度大。

### 方向E：KG辅助的偏好推理（基于MedRAG+K-RagRec的思想）
**难度**：中 | **创新性**：⭐⭐⭐⭐ | **实现可行性**：高

把用户的历史偏好编码成KG，新行为出现时在KG上推理——"用户这次选简洁模式，因为ta在写邮件场景下一直是简洁偏好"。不需要复杂的LLM调用。

### 方向F：NL遗忘的KG审计（基于HANKER的思想）
**难度**：中低 | **创新性**：⭐⭐⭐ | **实现可行性**：高

"删除上个月关于项目X的记录" → 解析出{时间、话题、动作} → 在KG上找到相关节点 → 逻辑删除。比简单的关键词匹配精确一个量级。

### 方向G：自适应遗忘曲线索引（MemoryBank提升版）
**难度**：低 | **创新性**：⭐⭐⭐ | **实现可行性**：极高

MemoryBank的艾宾浩斯遗忘曲线 + 三维评分（时间+频率+重要性）。你的TencentDB pipeline稍微改改就能实现。虽然创新性不是最高，但作为基础方案加上融合其他方向可以打组合拳。

## 四、推荐的组合策略

考虑到你的团队情况和比赛时间：

### 稳健方案（保一等冲特等）
```
时序KG记忆层（方向B） + NL遗忘的KG审计（方向F） + MemoryBank遗忘曲线（方向G）
```
三个方向互相补充，工程实现有现成基础（TencentDB可以改）。

### 激进方案（冲擂主）
```
Agent自主记忆管理（方向A/简化版） + 轻量RL检索选择器（方向C） + 时序KG（方向B）
```
这个组合创新性最强，但需要更早拿到麒麟SDK做benchmark。

### 性价比方案（稳拿奖）
```
KG辅助的偏好推理（方向E） + MemoryBank遗忘曲线（方向G） + NL遗忘审计（方向F）
```
工程实现难度最低，三个方向都有论文可参考，覆盖面广。
